Get up and running with Ascend on a docker. This guide will walk you through installation, configuration, and creating your first experiment.

## Prerequisites

### Hardware Requirements

#### Local Development

| Resource | Minimum |
|----------|---------|
| **RAM** | 8 GB |
| **CPU** | 4 cores |
| **Disk Space** | 20 GB free |
| **OS** | macOS 11+, Linux (Ubuntu 20.04+) |

### Software Requirements

The following tools must be installed on your system:

| Tool | Minimum Version | Installation |
|------|----------------|--------------|
| **Docker** | v20.10+ | [Install Guide](https://docs.docker.com/get-docker/) |
| **curl** | v7.68+ | Pre-installed on most systems |

:::note
The installation script will automatically check for these prerequisites and guide you through installing any missing tools.
:::


## Quick Start: One-Command Installation

Get Ascend running on a docker in under 10 minutes:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/**/install.sh)
```

## Troubleshooting


### Uninstallation

To remove Ascend from your cluster:

```bash
# Using the uninstall script
cd ascend
./uninstall.sh

```

## What's Installed

After successful installation, your cluster will have:

| Component | Purpose |
|-----------|---------|
| **ascend-testlab** | Backend Experiment service |
| **ascend-flockr** | Backend Audience service |
| **ascend-panel** | Frontend panel for Experimentation |
| **ascend-sdks** | SDKs for iOS, Android and RN Integration |
